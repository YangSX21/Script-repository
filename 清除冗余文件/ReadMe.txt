在终端运行脚本unuse.py。参数为项目地址，在项目的上级目录会生成一系列结果文档。

结果文档中“xx最后的文件"即最后出的结果，里面的文件还需要进行人工筛选，例如该文件中有定义被使用的宏，变量，该类有过重命名。脚本并没有检查出这种.把你认为还有用的文件名从该文档中删除。

结果文档中“Xx文件夹内的所有的pbxproj文件”，根据3中的筛选结果，把文件夹内不用不涉及的pbx文件地址也去掉，当然你也可以不去掉，这会影响下一个脚本运行速度。

运行脚本deletePhx，参数第一个是“Xx文件夹内的所有的pbxproj文件”，第二个是“xx最后的文件"，第三个参数是pbx文件备份的地址，第四个参数是“xx引用的所有文件”。
